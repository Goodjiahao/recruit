## 1. 项目目标
- 构建 Spring Boot 后端，连接本机 MySQL `bigdata` 库，读取已由 Spark 分析落库的 18 张 ADS 结果表。
- 通过 RESTful 接口（`GET /api/{board}/{metric}`）为 Vue3 + ECharts 大屏提供标准化数据。
- **配置驱动**：18 个指标共用一套通用接口，仅靠 `t_metric_meta` 配置表驱动，新增指标零新增业务代码。
- （预留）AI 智能问答模块 `/api/ai/chat`，按工作量决定是否实现。

## 2. 已确认技术决策
| 项 | 决策 |

| AI 模块 | 先预留接口与前端空面板，后续按工作量决定 |

## 6. 风险与对策
| 风险 | 对策 |
|---|---|
| 本机无 JDK 21 | 安装 JDK 21，或将 pom `java.version` 降到本机已有版本（如 17）并确认 Boot 4 兼容 |
| DDL 列名与模板不一致 | 以 Navicat 导出真实结构为准，生成数据层 / 前端字段映射 |
| 字段不统一 | 等全员 SQL 到齐后统一落库，接口按 `t_metric_meta` 动态读取，靠表名前缀天然隔离 |

## 7. 配置约定
- 数据源：`jdbc:mysql://<host>:3306/bigdata?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8`，user `root`，password `123456`
- 接口返回统一信封：`{code, msg, data:[...]}`
- 结果表命名：`ads_<板块>_<指标>`；接口路径：`/api/<板块>/<指标>`
